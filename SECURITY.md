# Security Policy

## Supported Versions

Unbotherd is actively developed, and security updates are provided for the **latest released version** available on Google Play.

Older versions may not receive security updates.

---

## Reporting a Vulnerability

If you believe you have discovered a **security vulnerability** in Unbotherd, please report it responsibly.

### Please do NOT:
- Open a public GitHub issue for security vulnerabilities
- Share exploit details publicly
- Include sensitive or personal data in reports

---

---

## Scope

Security reports may include issues related to:
- App locking behavior
- Notification handling
- Permission usage
- Local data storage
- Bypass or unintended access scenarios

Unbotherd operates **entirely on-device** and does not run backend servers.

---

## Responsible Disclosure

If a valid security issue is reported:
- We will acknowledge receipt within a reasonable timeframe
- We will investigate and address the issue as appropriate
- We may release a fix in a future app update

Public disclosure should wait until a fix has been released or coordinated.

---

## Bug Bounties

At this time, Unbotherd does **not** offer a formal bug bounty program.

However, responsible disclosures are appreciated and help improve the app for everyone.
